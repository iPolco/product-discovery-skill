# Шаблоны опросов для `ask_user_input_v0`

Готовые JSON-объекты для Шага 0. Не пересобирай структуру вопросов с нуля — бери отсюда и адаптируй под конкретный проект, если нужно.

Разбивка по группам: каждый вызов `ask_user_input_v0` содержит до 3 связанных вопросов, чтобы человек отвечал в потоке, а не 16 отдельными тапами. Открытые вопросы без вариантов ответа (название проекта, описание, конкуренты, бюджет, инвестиции) задавай обычным текстом — для них `ask_user_input_v0` избыточен.

## Первый вызов: рынок + тип аудитории + стадия

```
ask_user_input_v0(questions=[
  {
    "question": "Рынок проекта (страна/регион)?",
    "type": "single_select",
    "options": ["Россия", "СНГ", "EU/UK", "США", "Глобальный", "Другой"]
  },
  {
    "question": "B2B или B2C?",
    "type": "single_select",
    "options": ["B2C", "B2B SMB", "B2B Enterprise", "B2B2C"]
  },
  {
    "question": "Стадия проекта?",
    "type": "single_select",
    "options": ["Idea (ещё нет MVP)", "MVP готов, нет клиентов", "Есть первые клиенты", "Есть выручка"]
  }
])
```

## Второй вызов: тип бизнеса + горизонт + тип рынка

Задавай после того, как человек описал, что делает продукт (иначе сложно выбрать тип бизнеса).

```
ask_user_input_v0(questions=[
  {
    "question": "Тип бизнеса?",
    "type": "single_select",
    "options": [
      "Product (SaaS, приложение, физ. товар с установкой)",
      "Service / Консалтинг (продаётся время и экспертиза)",
      "Marketplace (двусторонняя платформа)",
      "Hardware (физ. устройство с производственным циклом)"
    ]
  },
  {
    "question": "Горизонт финансового планирования?",
    "type": "single_select",
    "options": ["12 месяцев", "24 месяца"]
  },
  {
    "question": "Тип рынка (если знаешь)?",
    "type": "single_select",
    "options": [
      "Существующий (конкуренты есть, клиенты знают проблему)",
      "Ресегментированный (новая ниша внутри рынка)",
      "Новый (клиенты ещё не осознают проблему)",
      "Не уверен — определим в задаче 1"
    ]
  }
])
```

## Третий вызов (если стадия ≠ idea): клиенты + CRM-данные

Для idea-стадии пропусти — этих данных нет.

```
ask_user_input_v0(questions=[
  {
    "question": "Первые клиенты или продажи есть?",
    "type": "single_select",
    "options": ["Нет", "Да, до 10", "Да, 10–50", "Да, 50–200", "Да, больше 200"]
  },
  {
    "question": "Есть ли данные из CRM / аналитики, которые можно приложить к PD?",
    "type": "single_select",
    "options": [
      "Да, приложу файл (CSV/Excel)",
      "Есть, но не готов делиться — использую экспертные оценки",
      "Нет, всё в голове"
    ]
  },
  {
    "question": "Проводился ли PD по этому проекту раньше?",
    "type": "single_select",
    "options": [
      "Да, приложу прошлый финплан .xlsx",
      "Да, но файла нет — начнём заново",
      "Нет, это первый раз"
    ]
  }
])
```

**Уточнение для маркетплейсов:** если пользователь озвучил количество клиентов одной цифрой («800 активных пользователей»), обязательно переспроси в свободной форме: «Это суммарно на обеих сторонах (поставщики + покупатели) или на каждую сторону отдельно?». Liquidity маркетплейса оценивается по **меньшей** стороне — 800 суммарно при перекосе 700/100 даёт liquidity 100, а это уже ниже порога инвест-трекера ≥ 20 на сторону.

## Четвёртый вызов: выбор режима + этап раунда (если релевантно)

Задавай после того, как собрал остальные данные — на основе стадии и целей можешь сам предложить подходящий режим.

```
ask_user_input_v0(questions=[
  {
    "question": "В каком режиме проводить PD?",
    "type": "single_select",
    "options": [
      "Light (~45 минут) — быстрая проверка идеи",
      "Full (~2–3 часа) — полный анализ для инвестора"
    ]
  },
  {
    "question": "Какой инвестиционный раунд планируете (если готовитесь)?",
    "type": "single_select",
    "options": [
      "Pre-seed / seed (ангел или ранний фонд)",
      "Series A",
      "Series B или позже",
      "Не планирую раунд / буду бутстрапить",
      "Пока не знаю"
    ]
  }
])
```

Этап раунда меняет пороги инвест-трекера в финальной сводке. Если человек выбрал «Pre-seed/seed» или «Не знаю» — скил использует дефолтные пороги (стандартные для early-stage). Если Series A — пороги качественно другие: ARR ≥ $1M, рост ≥ 10% MoM, NRR ≥ 110%, Payback ≤ 12 мес. Подробнее см. раздел «Трекер инвест-готовности» в SKILL.md.

## Вопросы в свободной форме (не через `ask_user_input_v0`)

Эти вопросы задаёшь обычным текстом — у них нет закрытого списка ответов:

1. Название проекта?
2. Что продукт делает? Какую проблему решает? (2–3 предложения)
3. Основные целевые сегменты? (кто клиент)
4. 2–3 основных конкурента (названия)
5. Бизнес-модель: подписка / комиссия / разовая продажа / freemium / другое?
6. Бюджет на тестирование гипотез, ₽ (используется в задаче 16)
7. Инвестиции (сумма в ₽ или «нет»)

## Пример общего потока

1. Читаю описание проекта от пользователя (из первого сообщения).
2. Делаю первый `ask_user_input_v0` (рынок + B2B/B2C + стадия).
3. Если в первом сообщении нет описания — спрашиваю в свободной форме: название, что делает, конкуренты.
4. Делаю второй `ask_user_input_v0` (тип бизнеса + горизонт + тип рынка).
5. Если стадия ≠ idea — делаю третий `ask_user_input_v0` (клиенты + CRM + прошлый PD).
6. Спрашиваю в свободной форме: бюджет, инвестиции.
7. Предлагаю режим на основе собранного (idea → Light, MVP+клиенты+seed → Full) и делаю четвёртый `ask_user_input_v0`.
8. Запускаю pre-flight check, инициализирую KB, подтверждаю план с человеком.

Итого: **4 вызова `ask_user_input_v0`** + несколько свободных вопросов вместо 16 отдельных. Пользователь отвечает быстро, не устаёт.

## Финальный шаг: зафиксируй режим в окружении

После того как пользователь выбрал режим Light или Full, ОБЯЗАТЕЛЬНО установи переменную окружения:

```bash
export PD_MODE=light   # или full
echo "PD_MODE=$PD_MODE"
```

Это защитит от двух сценариев ошибок:

1. **`delete_light_slides.py` в Full-режиме** — если запустить по ошибке, удалит 13 нужных слайдов (тренды, PESTEL, CJM, альтернативный сценарий, PMF). Скрипт проверяет `PD_MODE` и отказывается работать если режим не совпадает.
2. **Запуск без явного режима** — скрипты откажутся работать вообще, потребуют явно установить `PD_MODE`.

Проверь установку: `scripts/preflight_check.sh` выведет текущее значение `PD_MODE`.
